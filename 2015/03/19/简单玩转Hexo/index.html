简单玩转Hexo

简介

Hexo是一个很好的博客，个人觉得他的界面干净利落，阅读体验也是很好。初次接触Hexo是在朋友的介绍下才知道的，hexo是需要手动搭建（觉得B格很高），在朋友的帮助下也终于是完成了。
开篇记录一下搭建的过程希望能帮助到浏览到这里的朋友
我的系统为OS X，windows也可以参考。

hexo是一个基于Node.js的静态博客程序，可以方便的生成静态网页托管在github上。

怎样搭建hexo博客

–

安装Git

作用：把本地的hexo内容提交到github上。

安装Xcode就自带Git，这步可以忽略。

安装Node.js

作用：用来生成静态页面的。

到Node.js官网下载就OK。

安装Hexo

Node和Git都安装好后，可执行如下命令安装hexo:

npm install -g hexo

初始化

我们可以创建一个新文件夹作为hexo的安装目录，把所有的hexo文件都放里面，主要是为了方便管理。
然后，执行init命令初始化Hexo到你指定目录：（以下有两种方式初始化）

方式一：直接指定目录

hexo init folder
folder:是制定目录的文件路径。例：/Documents/hexo

方式二：进入指定目录（cd / 文件夹路径）

hexo init

终端执行成功后显示结果：

[info] Copying data
[info] You are almost done! Don't forget to run `npm install` before start blogging with Hexo!

安装依赖包

sudo npm install

生成静态页

cd 到你的hexo安装目录（工作目录），执行如下命令

hexo generate 简写 hexo g

必须到你新建的hexo安装目录下执行，否则不成功

生成静态网页，本地启动测试

hexo server 简写 hexo s

到此本地服务已经完成，可以再浏览器中输入http://localhost:4000进行查看。接下来需要把我们创建的静态页面托管到github上，别人才能访问到

问题

这里可能会出现一些问题导致本地的预览界面出不来

  • 问题1：

    Besides,utilities are separated into a standalone.hexo.util is not
reachable anymore.

    解决方法：（执行命令）

    npm install hexo-server --save
  • 问题2：

什么都没有显示或者显示Cannot GET/

解决方法，由于2.6以后就更新了，我们需要手动配置些东西，执行命令：

npm install hexo-renderer-ejs --save
或
npm install hexo-renderer-stylus --save
或
npm install hexo-renderer-marked --save

部署到Github账号

注册Github账号

这里就不再赘述，已有Github账号可以跳过此步骤。没有的点击官网去注册。

新建repository（仓库）

登陆Github账号后，点击右上角的“+”号按钮，选择“New repository”

在Create a new repository界面填写格式如下图所示: 用户名.github.io

填写完成点Create repository创建完成

生成SSH Keys：

我们如何让本地git项目与远程的github建立联系？这时候就要用到SSH Keys

生成SSH Keys

使用ssh-keygen命令生成密钥对

ssh-keygen -t rsa -C "这里是你申请Github账号时的邮箱"

然后系统会要你输入密码：（我们输入的密码会在你提交项目的时候使用）

Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>

（终端提示生成的文件路径）找到你生成的密钥找到id_rsa.pub用终端进入编辑，复制密钥。

添加你的SSH Key到ssh-agent

//在后台打开 ssh-agent
eval "$(ssh-agent -s)"

添加你的SSH Key到ssh-agent
ssh-add ~/.ssh/id_rsa

添加SSH Key到Github：

添加SSH Key

通过命令复制SSH Key内容到系统剪贴板

pbcopy < ~/.ssh/id_rsa.pub

登陆Github,点击右侧用户按钮，选择Settings

点击Add SSH Key 按钮， 将复制的秘钥粘贴到Key 栏

测试能不能连接成功

ssh -T git@github.com

执行结果

Permanently added the RSA host key for IP address '192.30.252.130' to the list of known hosts.
Are you sure you want to continue connecting (yes/no)?<输入yes>
Hi username! You've successfully authenticated, but GitHub does not

现在你已经可以通过SSH链接到Github了

如果有问题，请再配置。参考网站

部署

编辑 _config.yml(在你的工作目录下)，把下面的your_username换成你的github用户名

deploy:
      type: git 
      repo: https://github.com/your_username/    your_username.github.io.git
branch: master

执行部署命令：

hexo henerate
hexo deploy

  • 问题1

    部署时出现：Error: EACCES, open ‘/Users/Desktop/hexo/public/js/script.js’
    原因：权限问题在部署命令前加sudo

  • 问题2

    deployer找不到git: ERROR Deployer not found: git

    解决方法:npm install hexo-deployer-git –save

  • 问题3

    { [Error: Cannot find module './build/Release/DTraceProviderBindings'] code: 'MODULE_NOT_FOUND' }
{ [Error: Cannot find module './build/default/DTraceProviderBindings'] code: 'MODULE_NOT_FOUND' }
{ [Error: Cannot find module './build/Debug/DTraceProviderBindings'] code: 'MODULE_NOT_FOUND' }

    解决方法：
    npm install hexo --no-optional
